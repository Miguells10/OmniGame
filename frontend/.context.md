# 🧠 Frontend Context Mapper

## 🏗 Arquitetura Atual
- **Stack Core:** React 18, Vite (para empacotamento AOT e servidor de desenvolvimento proxyado).
- **Substrato de Tipagem:** TypeScript Ativado com checagem estática no frontend (Interfaces DTO-Espelho aplicadas).
- **Sistema de Estilização:** Tailwind CSS (versão 3.4+) injetado via processos PostCSS.
- **UX/Design System:** Tema próprio chamado **Nexus-Style Dark UI**. Foco central na estética "Black & Neon" gamificada, fontes hi-tech e modais *Glassmorphism* usando desfoque de cenário de tela.

---

## 📦 Estado dos Componentes e Subsistemas 

### 📁 Infraestrutura & Configuração Principal
- `[x]` `tailwind.config.js` (Repositório de tokens de design. Definições de paletas hexadecimais como "nexus-bg" e "accent-cyan". Declaração de keyframes para glows dinâmicos).
- `[x]` `index.css` (Criação nativa da regra Glass-Panel unindo rgba transparente ao backdrop-filter).
- `[x]` `vite.config.ts` (Configuração fundamental para que não haja CORS-Errors durante desenvolvimento, agindo como proxy reverso para `http://localhost:8080`).

### 📁 Casco Funcional & Roteamento (Views & Layouts)
- `[x]` `AppShell.tsx` (Esqueleto base: Top Navigation persistente e Sidebar de rápido acesso adaptativo/colapsável).
- `[x]` `GameDashboard.tsx` (Página de destino (Hub). Carrega o Grid com os Cards Interativos e chips de filtros base do catálogo do EAV backend).

### 📁 Features Centrais (Widgets)
- `[x]` `AiChatWidget.tsx` (**The Collector**): Componente mais complexo da arquitetura de vista. Representa a Interface Conversacional da plataforma. Retém e manipula um estado em memória de "Diálogo RAG". Consome explicitamente a API de Streaming (Server Sent Events) escrevendo texto na tela caractere por caractere (Efeito LLM natural).

### 📁 Clientes HTTP e Integradores
- `[x]` `api.ts` (Envelopamento completo da base URL da aplicação, central de disparo de payloads e injeção automática em Headless mode de tokens de autenticação `Authorization: Bearer <token>`).
- `[x]` `types.ts` (Armazena as interfaces primárias `Game`, `GameEntity`, manipuladoras para que requisições ao Back Java não cheguem genéricas tipo `any`).
